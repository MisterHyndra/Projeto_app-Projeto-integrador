Guia de Instalação e Configuração

Este guia irá ajudá-lo a configurar o ambiente de desenvolvimento do Med Alert em sua máquina local.

Pré-requisitos

Antes de começar, certifique-se de ter instalado em sua máquina:

  • Node.js (versão 16.x ou superior)
  • npm (geralmente vem com o Node.js) ou Yarn
  • Git para controle de versão
  • PostgreSQL (versão 12 ou superior)

1 Clonar o repositório

Comece clonando o repositório do projeto:

# Clonar o repositório
git clone https://github.com/seu-usuario/med-alert.git

# Navegar até o diretório do projeto
cd med-alert

2 Instalar dependências

Instale as dependências do projeto:

# Instalar dependências do backend
cd backend
npm install

# Instalar dependências do frontend
cd ../frontend
npm install

3 Configurar variáveis de ambiente

Crie e configure os arquivos de ambiente necessários:

Backend (.env)

Crie um arquivo .env na pasta backend com as seguintes variáveis:

# Configurações do servidor
PORT=3000
NODE_ENV=development

# Configurações do banco de dados
DB_HOST=localhost
DB_PORT=5432
DB_NAME=medalert
DB_USER=seu_usuario
DB_PASSWORD=sua_senha

# Chave secreta para JWT
JWT_SECRET=sua_chave_secreta_aqui
JWT_EXPIRES_IN=7d

# Configurações de e-mail (opcional, necessário para notificações por e-mail)
EMAIL_HOST=smtp.exemplo.com
EMAIL_PORT=587
EMAIL_USER=seu_email@exemplo.com
EMAIL_PASSWORD=sua_senha
EMAIL_FROM="Med Alert <noreply@medalert.com>"

# Configurações de notificação
NOTIFICATION_REMINDER_INTERVAL=15 # minutos entre lembretes
MAX_REMINDERS=3 # número máximo de lembretes

Frontend (.env)

Crie um arquivo .env na pasta frontend com as seguintes variáveis:

# URL da API
REACT_APP_API_URL=http://localhost:3000

# Configurações do Firebase (para notificações push)
REACT_APP_FIREBASE_API_KEY=sua_chave_aqui
REACT_APP_FIREBASE_AUTH_DOMAIN=seu-projeto.firebaseapp.com
REACT_APP_FIREBASE_PROJECT_ID=seu-projeto
REACT_APP_FIREBASE_STORAGE_BUCKET=seu-projeto.appspot.com
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=123456789012
REACT_APP_FIREBASE_APP_ID=1:123456789012:web:abcdef123456
REACT_APP_FIREBASE_MEASUREMENT_ID=G-ABCDEF1234

4 Configurar o banco de dados

Siga estes passos para configurar o banco de dados PostgreSQL:

Criar o banco de dados

Conecte-se ao PostgreSQL e execute:

CREATE DATABASE medalert;
CREATE USER seu_usuario WITH PASSWORD 'sua_senha';
GRANT ALL PRIVILEGES ON DATABASE medalert TO seu_usuario;
\c medalert
GRANT ALL ON SCHEMA public TO seu_usuario;

Executar migrações

Execute as migrações para criar as tabelas:

cd backend
npx sequelize-cli db:migrate

Popular o banco de dados (opcional)

Se desejar, você pode popular o banco com dados de exemplo:

npx sequelize-cli db:seed:all

5 Iniciar os servidores

Backend

Em um terminal, inicie o servidor backend:

cd backend
npm run dev

Frontend

Em outro terminal, inicie o servidor de desenvolvimento do frontend:

cd frontend
npm start

O aplicativo estará disponível em http://localhost:3001 (a porta pode variar).

Solução de problemas

Erro ao conectar ao banco de dados

Verifique se o PostgreSQL está em execução e se as credenciais no arquivo .env estão corretas.

Erro de permissão no PostgreSQL

Certifique-se de que o usuário do banco de dados tem as permissões necessárias. Consulte a documentação do PostgreSQL para configurar as permissões corretamente.

Erro de porta em uso

Se a porta 3000 já estiver em uso, você pode alterar a porta no arquivo .env do backend.

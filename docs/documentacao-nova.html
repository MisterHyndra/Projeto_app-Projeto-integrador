Visão Geral

O Med Alert é um sistema completo para gerenciamento de medicamentos, projetado para ajudar usuários a manterem o controle de seus tratamentos médicos com lembretes inteligentes e acompanhamento detalhado.

Status Inteligente

Acompanhamento em tempo real do status de cada medicamento (agendado, tomado, perdido).

Notificações

Sistema avançado de lembretes com múltiplos canais de alerta.

Relatórios

Histórico detalhado e relatórios de adesão ao tratamento.

Banco de Dados

PostgreSQL

Utilizamos o PostgreSQL como banco de dados relacional principal, com o pgAdmin 4 para gerenciamento.

Configuração:

  • Host: process.env.DB_HOST
  • Porta: process.env.DB_PORT
  • Banco: process.env.DB_NAME
  • Usuário: process.env.DB_USER

Modelo de Dados

Tabela Descrição Campos Principais
users Dados dos usuários id, name, email, password_hash, created_at, updated_at
medications Cadastro de medicamentos id, user_id, name, dosage, frequency, time_of_day, days_of_week, status, start_date, end_date, active
medication_logs Histórico de medicações id, medication_id, user_id, taken_at, status, notes
notifications Agendamento de notificações id, user_id, medication_id, scheduled_for, status, notification_type

Sistema de Notificações

Fluxo de Notificações

1

Notificação Inicial

Enviada no horário programado do medicamento.

Status: Agendado Ação: Exibe notificação
2

Primeiro Lembrete

Enviado 15 minutos após a notificação inicial se não houver interação.

Status: Pendente Ação: Notificação com som/vibração
3

Segundo Lembrete

Enviado 30 minutos após a notificação inicial se não houver confirmação.

Status: Atrasado Ação: Notificação com som/vibração

Notificação de Emergência

Após 1 hora sem confirmação, o sistema envia um e-mail para os contatos de emergência cadastrados.

Status: Crítico Ação: E-mail para contatos de emergência

Configuração de Notificações

  • Tipo de Alerta

    Personalizável por usuário (som, vibração ou ambos)

  • Intervalos de Lembrete

    Configurável (padrão: 15min, 30min, 1h)

  • Notificações por E-mail

    Alertas para contatos de emergência

Status de Medicamentos

Agendado

Status inicial de todo medicamento cadastrado. Indica que o medicamento está programado para ser tomado no horário definido.

Tomado

Status atribuído quando o usuário confirma que tomou o medicamento. Pode ser feito manualmente ou através da notificação.

Perdido

Status atribuído automaticamente quando o usuário não confirma o medicamento após todos os lembretes.

Fluxo de Atualização de Status

  1. 1. Cadastro do Medicamento

    Status definido como agendado por padrão.

  2. 2. Horário do Medicamento

    Notificação inicial é enviada ao usuário.

  3. 3. Lembretes

    Até 3 lembretes são enviados em intervalos configuráveis.

  4. 4. Confirmação

    Usuário confirma que tomou o medicamento, atualizando para tomado.

  5. 5. Não Confirmação

    Se não houver confirmação após todos os lembretes, o status muda para perdido e os contatos de emergência são notificados.

Arquitetura do Sistema

O Med Alert foi desenvolvido com uma arquitetura moderna, separando claramente o frontend móvel do backend em Node.js.

Frontend Móvel

  • Desenvolvido com React Native
  • Interface responsiva e acessível
  • Context API para gerenciamento de estado
  • Navegação com React Navigation
  • Notificações locais com expo-notifications

Backend

  • Node.js com Express
  • Banco de dados PostgreSQL
  • Autenticação JWT
  • API RESTful
  • Serviço de e-mail para notificações

Modelo de Dados

  • Usuários (autenticação e perfil)
  • Medicamentos (nome, dosagem, frequência)
  • Histórico de medicações
  • Contatos de emergência
  • Configurações de notificação

Tecnologias Utilizadas

Frontend

  • React Native Framework para desenvolvimento mobile
  • React Navigation Navegação entre telas
  • Expo Notifications Sistema de notificações push

Backend

  • Node.js Ambiente de execução JavaScript
  • PostgreSQL Banco de dados relacional
  • Nodemailer Envio de e-mails para contatos de emergência

Guia de Instalação e Configuração

Siga este guia para configurar o ambiente de desenvolvimento do Med Alert em sua máquina local.

# Clonar o repositório
git clone https://github.com/seu-usuario/med-alert.git
cd med-alert

# Instalar dependências
npm install

# Configurar variáveis de ambiente
cp .env.example .env
# Editar o arquivo .env com suas configurações

# Iniciar o servidor de desenvolvimento
npm run dev
